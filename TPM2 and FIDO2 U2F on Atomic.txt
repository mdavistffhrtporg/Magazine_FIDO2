As previously discussed, there options for unlocking LUKS partitions by using systemd-cryptenroll in coordination with a TPM2 chip and FIDO2 U2F security keys. By utilizing this, TPM2 chips and FIO2 U2F security keys offer an alternative to entering a password every time the system is booted. On other non-atomic editions, this is pretty straight forward. But, there is no consistent documentation on how to do this on Fedora Atomic Desktops.
